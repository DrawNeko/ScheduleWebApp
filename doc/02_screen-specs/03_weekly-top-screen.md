⑥トップ画面仕様（週間表示）

【対象画面】
- index.html

【目的】
- 選択中グループのメンバーに対し、1週間分の予定を俯瞰表示する。

【UI要件】
1. ヘッダ右上にハンバーガーメニューを表示し、先頭にログインユーザ情報（ユーザ名 / ユーザID / ロール名）を3行で表示する。
2. 表示グループ選択プルダウンと「編集」ボタンを表示する。
3. 日付移動ボタン（1日前/1日後/1週前/1週先/今週）を表示する。
4. 週間スケジュール表（メンバー行 × 7日列）を表示する。
5. 日付ヘッダには西暦を含めた形式（YYYY/M/D）を表示する。
6. 日付ヘッダの文字色は、土曜日を青・日曜日を赤で表示する。
7. 土日のセル背景色はグレーで表示する。
8. 予定チップを表示し、色設定を反映する。

【操作仕様】
- 予定チップのダブルクリックで編集画面へ遷移する。
- 空白セルのダブルクリックで新規登録画面へ遷移する。
- グループ変更時は対象メンバー/予定を再取得して再描画する。
- 「デフォルトに設定」押下で表示グループを保存する。
- 「編集」押下でグループ編集画面（`/group_manage.html`）へ遷移する。

【データ取得】
- グループ一覧: `GET /api/groups`
- グループ所属ユーザ: `GET /api/groups/:groupId/users`
- 予定一覧（期間指定）: `GET /api/schedules?start=YYYY-MM-DD&end=YYYY-MM-DD&include_users=1`

【描画ルール】
- 表示期間は基準日の週（7日間）とする。
- 予定は日付・参加者に応じて対応セルへ配置する。
- 参加者情報は schedule と user_schedule の関連で解決する。

## ユーザメンテナンス導線

- トップ画面ヘッダのハンバーガーメニューに以下項目を上から表示する。
  1. ユーザ情報（ユーザ名 / ユーザID / ロール名）
  2. ユーザ登録
  3. ユーザ情報編集（ログインユーザ自身）
  4. ユーザメンテナンス（権限付与 / ユーザ削除 / ※adminのみ他ユーザのパスワードリセット）
  5. ログアウト
- ロールに応じて利用不可機能のメニュー項目は非表示とする。
- ユーザメンテナンスでは、自身と同等以上の権限ユーザに対する権限変更・削除・パスワードリセットを不可とする。
- ユーザメンテナンスのロール欄プルダウンは、ログインユーザが付与可能な項目のみ表示し、「開発者（admin）→ リーダー（ld）→ 一般（normal）→ BP（business）」の固定順で表示する。
- ユーザメンテナンス画面では、ユーザID/ユーザ名/メールの部分一致検索とロール一致検索でリアルタイム絞り込みを行える。
- ユーザ情報編集ではメールアドレスとパスワードを更新できる。
- ユーザ情報編集のパスワード欄は表示/非表示を切り替えできる。
