# バリデーション仕様

## 1. 対象範囲

- ログイン画面（`login.html`）
- スケジュール登録画面（`schedule_add.html`）
- スケジュール編集画面（`schedule_edit.html`）
- ユーザ登録画面（`user_register.html`）
- サーバサイド（定期予約の最低限チェック）

## 2. 画面別バリデーション

### 2.1 ログイン画面

| 項目 | ルール | エラー時挙動 |
|---|---|---|
| ユーザID | 必須（`required`） | ブラウザ標準検証 |
| パスワード | 必須（`required`） | ブラウザ標準検証 |
| 認証 | `/api/auth/login` 成功必須 | 「ユーザIDまたはパスワードが正しくありません。」を表示 |

### 2.2 スケジュール登録画面

#### 必須項目

| 項目 | ルール |
|---|---|
| 登録者（`created_by`） | 必須 |
| タイトル | 必須 |
| 予定日 | 必須 |
| 開始時刻（時・分） | 必須 |
| 終了時刻（時・分） | 必須 |

#### 業務ルール

| ルール | 条件 | エラー文言 |
|---|---|---|
| 時刻整合 | `終了時刻 >= 開始時刻` | `終了時刻は開始時刻以降を選択してください` |
| 定期予約終了日 | 定期予約ON時は必須 | `定期予約の終了日は必須です` |
| 定期予約日付整合 | `終了日 >= 予定日` | `定期予約の終了日は予定日以降にしてください` |

### 2.3 スケジュール編集画面

| 種別 | ルール | エラー時挙動 |
|---|---|---|
| 対象ID | `id` クエリ必須 | アラート後トップ遷移 |
| 時刻整合 | 終了時刻入力時のみ `終了>=開始` を検証 | 登録画面と同一文言で警告 |
| 定期予約終了日 | 定期予約ON時は必須・開始日以降 | 登録画面と同一文言で警告 |
| 削除範囲UI | 定期予定時のみ「定期予約ごと削除」表示 | OFF=対象日のみ / ON=シリーズ削除 |

### 2.4 ユーザ登録画面

| 項目 | ルール | エラー時挙動 |
|---|---|---|
| ユーザID | 必須 | `ユーザIDは必須です` |
| ユーザ名 | 必須 | `ユーザ名は必須です` |
| メール形式 | 入力時のみ正規表現チェック | `メールアドレスの形式が不正です` |
| ユーザID重複 | `/api/users/{user_id}` で既存確認 | `このユーザIDは既に登録されています` |

メール形式チェックに使用している正規表現:

```regex
^[\w\.-]+@[\w\.-]+\.\w+$
```

## 3. サーバサイド補足（定期予約）

- `is_recurring` が有効な場合、以下が必須
  - `recurring_end_date`
  - `recurring_frequency`
- `recurring_end_date < 開始日` の場合はエラー
- 現状、入力エラーは詳細な `400` 分岐ではなく、主に `500` で返却される

## 4. 今後の改善余地

- 入力エラーを `400 Bad Request` として明示的に返却
- フィールド単位エラー（例: `{ field: "title", message: "..." }`）の標準化
- フロントでのアラート表示から画面内エラーメッセージ表示への移行
