⑪ユーザ登録仕様

【対象画面】
- user_register.html

【目的】
- 新規ユーザ情報を登録し、ログイン可能な利用者を追加する。

【UI要件】
1. ユーザID、ユーザ名、メール、役職、パスワード入力欄を表示する。
2. 役職プルダウンはログインユーザのロールに応じて選択可能な項目のみ表示する（admin: 全ロール、ld: ld/normal/business、normal: normal/business）。
3. 役職プルダウンの表示順は「開発者（admin）→ リーダー（ld）→ 一般（normal）→ BP（business）」の固定順とする。
4. 役職プルダウンは角丸デザインで表示する。
5. 登録ボタン押下でユーザ作成APIを呼び出す。
6. 登録/キャンセルボタンのラベルは縦中央で表示する。
7. 失敗時はエラー表示、成功時は完了通知を表示する。

【入力制約】
- ユーザIDは一意であること。
- 必須項目（ユーザID、ユーザ名、パスワード）を満たすこと。

【API】
- `GET /api/users/roles?assignable_for_create=1`: 登録画面用ロール一覧（ログインユーザの権限でフィルタ済み）
- `POST /api/users`: ユーザ登録
- 必要に応じて `GET /api/users`: 一覧取得

【補足】
- パスワードはサーバー側で適切に扱う（平文保存しない運用が望ましい）。
