⑧スケジュール編集仕様

【対象画面】
- schedule_edit.html

【目的】
- 既存スケジュールの内容・参加者・定期設定を更新する。
- 定期予定編集時は「定期予約を設定する」のON/OFFで変更範囲を切り替える。

【UI要件】
1. 既存値（タイトル/日時/場所/メモ/色/参加者）を初期表示する。
2. recurring_rule_id がある場合は定期設定をONで表示し、終了日/頻度を初期表示する。
3. 「定期予約を設定する」がOFFの場合、
   当日のみ変更となり、元の定期予約との同期が解除される旨の注釈を表示する。
4. 更新ボタン押下でバリデーション後に更新APIを呼び出す。
5. 更新成功後はトップ画面（/index.html）へ遷移する。

【入力制約】
- 終了時刻は開始時刻以上。
- 「定期予約を設定する」がONの場合は終了日が必須で、予定日以降。

【送信仕様】
- `PUT /api/schedules/:id` を呼び出す。
- 参加者は作成者 + 同報者を再構成して保存する。
- 変更範囲は `is_recurring` の値で判定する。
  - `is_recurring=true`: 定期予約全体を変更
  - `is_recurring=false`: 当日のみ変更（同期解除）

【サーバー処理ルール（要点）】
1. 単発→単発更新
   - 当該 schedule_id を更新
2. 単発→定期更新
   - recurring_rules を新規作成
   - 展開 schedules を作成
   - 元の単発 schedule を削除
3. 定期→当日のみ更新（is_recurring=false）
   - 対象 schedule_id のみ更新
   - 対象 schedule は recurring_rule_id を外して単発化（同期解除）
   - ルール配下の残件が0件になった場合は recurring_rules を削除
4. 定期→定期更新（is_recurring=true）
   - 既存ルール配下の実在スケジュールを基準に更新し、過去に単体削除された日付は再作成しない
   - recurring_rules を更新
   - 新ルール内容で schedules を再展開（単体削除済み日付は除外）
   - シリーズの開始日は既存ルールの start_date を維持する（途中日の編集でも開始日以前は欠落しない）
