# システム概要

## 1. 目的

- メンバーのスケジュールを共有・管理するWebアプリ
- ログインで利用者を識別し、表示グループ（Public / Private）単位で表示対象を切り替える

## 2. 技術スタック

| 区分 | 技術 |
|---|---|
| フロントエンド | HTML / CSS / JavaScript（ES Modules） |
| バックエンド | Node.js + Express |
| データベース | MySQL |

## 3. 主な機能

### 3.1 ログイン

- ユーザID + パスワードで認証
- ログイン後に `index.html` へ遷移
- 未ログインで保護ページにアクセスした場合は `login.html` に誘導

### 3.2 トップ画面（週間表示）

- 7日間のスケジュールを「メンバー行 × 日付列」で表示
- 1日前 / 1日後 / 1週前 / 1週先 の移動
- 表示グループ選択とデフォルト保存
- 予定チップのダブルクリックで編集、空白セルダブルクリックで新規登録

### 3.3 表示グループ

- Publicグループ: 全ユーザが選択可能
- Privateグループ: 作成者（owner）のみ選択可能
- デフォルト未設定時は先頭Publicグループを自動設定

### 3.4 スケジュール登録/編集

- 登録者・同報者（複数）を指定可能
- 終了時刻は開始時刻以降のみ有効
- 定期予約対応
  - 期間: 開始日は予定日、終了日は指定日
  - 頻度: `DAILY`（平日のみ）/ `WEEKLY` / `BIWEEKLY`

### 3.5 スケジュール削除

- 編集画面で削除確認モーダルを表示
- 「タイトル / 日時 / 場所」を確認可能
- 定期予定では削除範囲を選択
  - OFF: 対象日のみ削除
  - ON: 同一ルール配下を削除

## 4. APIカテゴリ

| カテゴリ | エンドポイント |
|---|---|
| 認証 | `/api/auth` |
| グループ | `/api/groups` |
| スケジュール | `/api/schedules` |
| マスタ | `/api/users`, `/api/colors` |
